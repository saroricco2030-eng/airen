# 앱 개발 공통 수칙

> **AI 행동 원칙**
> - 이 파일의 수칙을 어기는 코드·설계를 발견하면 즉시 알리고 수정 방향을 제안한다
> - 모호한 요청은 구현 전에 의도를 확인한다
> - 선택지가 필요한 경우 근거와 함께 제시하고 결정을 기다린다
> - 잘못된 방향이라 판단되면 솔직하게 피드백한다

---

## 0. 개발 워크플로우

모든 기능 요청은 아래 순서를 따른다.

```
1. PLAN      — 요구사항 분석 · 사용자 흐름 확인 · 경쟁앱 레퍼런스 검토
2. IMPLEMENT — 완성형 코드 제공 (바로 실행 가능한 상태)
3. VALIDATE  — 수칙 준수 여부 자체 점검 · 개선 포인트 리포트
```

- 파일 구조 변경 · 대규모 리팩토링은 반드시 사전 확인 후 진행
- 에러 발생 시: 원인 + 해결책을 함께 보고
- 빌드 완료 시: Post-Build Review(섹션 5) 자동 실행

---

## 1. 다국어 (i18n)
- 모든 앱은 다국어 지원을 전제로 설계한다
- 문자열은 처음부터 i18n 키로 코딩하며, 하드코딩 금지
- 기본 지원: 한국어(ko) · 영어(en) / 추가 언어는 프로젝트별 결정
- **AI 행동:** 하드코딩된 문자열 발견 시 즉시 알리고 i18n 키로 교체 제안

## 2. 보안
- 설계 단계부터 보안을 반영한다 (Security by Design)
- 필수 체크리스트
  - API 키 · 민감정보: 환경변수 또는 비밀 저장소 사용, 코드 직접 삽입 금지
  - **사용자 API 키는 localStorage에 저장하되 민감 데이터임을 명시**
  - 입력값 검증: 모든 사용자 입력은 서버·클라이언트 양측 검증
  - 인증/인가: 최소 권한 원칙 적용
  - 데이터 전송: HTTPS 강제
  - **외부 AI API 호출은 Next.js API Route를 통해서만 수행 (클라이언트 직접 호출 금지)**
- **AI 행동:** 보안 체크리스트 미충족 항목 발견 시 구현 전 경고

## 3. UX · 사용성
- 직관적 사용성을 최우선으로 한다
- 기준: Krug "Don't Make Me Think" 원칙, Nielsen 10 Heuristics
- 신규 기능 추가 시 사용자 흐름(User Flow) 먼저 설계 후 구현
- **AI 행동:** 3단계 이상의 탭/클릭이 필요한 흐름은 단축 방법을 함께 제안

## 4. 디자인 · 트렌드
- 레이아웃·색감은 빌드마다 최신 트렌드를 반영하여 변화를 준다 (획일적 금지)
- 트렌드 참조 소스
  - 앱 레퍼런스: Mobbin, Dribbble, App Store 피처드 앱
  - 전문가: Schoger · Malewicz · Gary Simon 및 해당 분야 유튜버
  - 학술: CHI · CSCW 등 HCI 컨퍼런스, UX 저널
- 동일 프로젝트 내에서도 기능 영역별로 시각적 변화 허용
- **AI 행동:** 디자인 결정 시 [참조 소스] → [적용 근거] → [구현 방법] 형식으로 명시

## 5. 완성도 평가 (Post-Build Review)
- 최초 빌드 완료 직후 자동으로 아래 항목 평가 후 리포트 제출
  - 저명 앱 리뷰어 시점 (App Store 에디터, The Verge, 9to5Mac 등 기준)
  - App Store 심사 가이드라인 적합성
  - 접근성 (WCAG 2.1 AA 이상)
  - 성능 (Lighthouse 기준)
  - 본 수칙 전체 항목 준수 여부
- 리포트 형식: `[항목] 현재 상태 → 개선 제안`
- 개선 여부는 사업주 판단으로 결정

## 6. 경쟁앱 분석 · 기능 제안
- 빌드 전: 동종업계 전세계 경쟁앱 기능 분석 → 차별화 포인트 제안
- 빌드 후: 경쟁앱 대비 부족한 기능 적극 제안
- 제안 형식: `[경쟁앱 기능] → [우리 버전 개선 방향] → [예상 효과]`
- **AI 행동:** 기능 구현 시 관련 경쟁앱 사례를 1~2개 함께 언급

## 7. 스토어 등록 기준
- 웹앱이므로 브라우저 접근 기준으로 하되 PWA 등록 가능성을 염두에 둔다
- 필수 반영 사항
  - 개인정보 처리방침 · 이용약관 페이지 포함
  - 사용자 API 키 수집 관련 안내 명시
- **AI 행동:** 심사 거절 가능성이 있는 코드·설계 발견 시 즉시 경고

## 8. 버전 관리
- 브랜치 전략: `main` (배포) · `dev` (개발) · `feature/기능명` (기능 단위)
- 커밋 컨벤션: `[type] 간단한 설명` 형식 사용
  - `feat`: 새 기능
  - `fix`: 버그 수정
  - `design`: UI/디자인 변경
  - `refactor`: 리팩토링
  - `chore`: 설정·의존성 변경
- 배포 전 반드시 `dev → main` PR 후 머지
- 태그 규칙: `v1.0.0` (major.minor.patch)

## 9. 파비콘 · OG 이미지
- 파비콘: 보라 계열 배경에 **AI**ren 로고타입 기반
- OG 이미지 (1200×630px): SNS 공유 시 브랜드 노출 목적
- 다크모드 파비콘 별도 대응

## 10. 에러 추적 · 크래시 리포팅
- 웹앱: **Sentry** 사용
- 로그 레벨 구분: DEBUG · INFO · WARNING · ERROR / 프로덕션에서 DEBUG 출력 금지
- 사용자 식별 정보는 로그에 포함하지 않는다 (개인정보 보호)
- **AI 행동:** 프로덕션 빌드에 DEBUG 로그 또는 콘솔 출력 잔존 시 즉시 알림

# 프로젝트 정보

## 기본 정보
- 앱 이름: **Airen** (**AI**ren — AI 강조 로고타입)
- 플랫폼: 웹앱 (Next.js) / PWA 고려
- 타겟 사용자: AI 도구를 업무에 활용하는 개인 사용자
- 주요 기능 요약:
  1. AI 모델 등록 (API 키 입력 → localStorage 저장)
  2. 질문 입력 → 등록된 모든 AI에 병렬 호출
  3. 카드형 답변 비교 뷰 (스트리밍 표시)
  4. 문단/섹션 선택 → 최종 편집창 조합
  5. 프롬프트 템플릿 (전문가 빙의, 교차검증 등 원클릭)
  6. 질문 히스토리 (localStorage)

## 경로
- 주요 소스 경로: `src/`
- i18n 파일 경로: `src/locales/ko.json`, `src/locales/en.json`
- 환경변수 파일: `.env.local` (Sentry DSN 등 서버 설정만 / 사용자 API 키는 localStorage)
- AI API 호출: `src/app/api/` (Next.js API Route)

## 디자인 시스템
- 주 색상: Purple `#7C3AED` (Violet-600)
- 보조 색상: Purple Light `#A78BFA` / Dark BG `#0D0D0F`
- 강조: **AI** 부분만 보라 계열 컬러/굵기로 차별화
- 폰트: Geist (Next.js 기본) — 영문, 한글은 Pretendard
- 다크모드: 기본 다크 / 라이트모드 추후 추가
- 컴포넌트 스타일: Glassmorphism 카드 (backdrop-blur + 반투명 보더)

## 외부 서비스 · 의존성
- Sentry (에러 추적)
- Vercel (배포 + 환경변수 관리)
- 각 AI 공급사 API (사용자 키 기반)

## 특이사항 · 주의사항
- **AI API 호출은 반드시 Next.js API Route 경유** (클라이언트에서 직접 외부 API 호출 금지)
- 사용자 API 키는 localStorage에 저장하며 서버로 전송 시 해당 요청에만 사용 후 서버 메모리에서 즉시 폐기
- 병렬 호출 시 일부 모델 실패해도 나머지는 정상 표시 (Partial failure 허용)
- 스트리밍 응답 지원 모델은 스트리밍으로, 미지원 모델은 완료 후 표시
